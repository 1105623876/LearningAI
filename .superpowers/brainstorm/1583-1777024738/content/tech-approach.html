对于概念可视化，你更倾向于哪种技术实现？

考虑开发效率和学习效果的平衡

A

交互式动画（高保真）

像 Distill.pub 或 3Blue1Brown 那样的流畅动画，可以拖动参数实时看效果

优点：直观性最强，理解深刻
缺点：开发周期长，每个知识点需要定制动画
技术：React + D3.js / Manim.js
B

静态图解 + 代码联动

精美的 SVG 图示，点击图中元素高亮对应代码，左右对照学习

优点：开发效率高，易于维护和扩展
缺点：交互性较弱，无法看到动态过程
技术：React + SVG + Monaco Editor
C

可执行代码 + 可视化输出

类似 Jupyter Notebook，修改代码立即看到矩阵变化、注意力热力图等

优点：动手实践，加深理解
缺点：需要后端支持代码执行，或用 Pyodide（较慢）
技术：FastAPI + Pyodide / JupyterLab
D

混合方案（推荐）

MVP 先做静态图解+代码联动（快速覆盖所有知识点），后续对核心概念（如注意力机制）补充交互动画

优点：平衡效率和效果，可迭代
缺点：需要规划好架构以支持多种可视化类型
